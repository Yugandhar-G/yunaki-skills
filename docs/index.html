The loop

How it works

No fine-tuning. No prompt babysitting. Yunaki watches what the agent does, distills the lesson, and feeds it forward.

1

Run & observe

The agent attempts a coding task. It fails, partially succeeds, or passes. The full execution trace and eval score are captured.

2

Extract the skill

Yunaki analyzes the trace and distills a reusable skill — when to apply it and the concrete steps that worked, stored in the skill bank.

3

Inject & improve

On the next task, relevant skills are retrieved semantically and injected into the prompt. Same model, measurably better output.

What's inside

Built like a real system, not a demo

Every piece of the loop is a first-class component with governance, retrieval, and evaluation baked in.

Self-Evolving Skills

Skills aren't static. They version, merge, and improve from each new execution that touches them.

Semantic Retrieval

Local embeddings plus regex triggers surface the right skill at the right moment — task-level or event-driven.

Skill Governance

Draft → pending → approved → active. Review and approve or reject before a skill ever reaches production.

Multi-Model Support

Gemini, Llama, or any OpenAI-compatible endpoint. The skill layer is model-agnostic by design.

LLM-as-Judge Eval

Scoring goes beyond pytest pass rates — an LLM judge grades quality so improvement is measured, not assumed.

Real-Time Dashboard

WebSocket-powered live runs. Watch scores climb and skills get created as the agent works.
