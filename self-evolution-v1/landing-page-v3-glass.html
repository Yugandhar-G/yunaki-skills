How it works

Fail once. Learn forever.

Six steps, one loop. Real tests grade every attempt — failures become reusable skills.

1

Run

Your agent attempts the task. Yunaki detects your CLI and drives it.

2

Score

pytest plus an optional LLM judge grade it. Real pass/fail, not vibes.

3

Extract

On failure, mine the trace for a reusable skill — single-trace or contrastive.

4

Bank

Store it with embeddings, ranking, and history in the skill bank.

5

Inject

Pull the right skills — semantic and score-weighted — into the next run.

6

Measure

Score again and report skill_delta against a no-skills control.

The honest metric

We won't lie about whether it worked.

Score-after-minus-before takes credit for everything that would've worked anyway. Yunaki runs a no-skills control arm and reports the honest delta. When it's small, we say so.

Engineers get honesty they can trust. Leaders get a real measurement system — not a dashboard anyone can game.

FAQ

Straight answers.

Does it work with my agent? +

Yes. Yunaki auto-detects your coding-agent CLI and drives it. If none is found, it falls back to a built-in Gemini SDK agent. You don't switch tools.

What's the control arm, and why trust the numbers? +

Before measuring improvement, Yunaki runs the same task with no skills injected — the control. We report skill_delta, the gain over that baseline, never raw score-after-minus-before. When it's small, we show it anyway.

Where are skills stored? +

In a MongoDB-backed skill bank — skills, history, and embeddings — alongside your runs and evaluations. Ranking, merge/drop, and history are first-class.

Do I need a Gemini API key? +

Only as a fallback — if no coding-agent CLI is detected, or if you pin a Gemini model for meta-operations. With a detected CLI, it's optional.

Is my code leaving my machine? +

Yunaki runs locally against your repo and your chosen agent backend. You control the backend, storage, and auth posture.
