§01
Agents repeat your repo's mistakes.

They don't know your conventions, your gotchas, the decision in PR #42. The usual fix — have an LLM rewrite the skill's prompt — measurably makes it worse. So we never touch the skill. We evolve what it knows.

−4.2ppLLM-rewriting a skill
+16.2pphuman-curated (SkillsBench)
−1.3ppself-generated (SkillsBench)

rewriting degrades. curation compounds. so evolve the context, not the method.

§02
The skill stays fixed. The memory evolves.

A skill is its method plus what it has learned about your repo. The whole system is one loop — and the skill's text is never part of what changes:

on invocation

recall.py runs at skill-load and inlines the skill's current repo context above the method — deterministic, skill-scoped.

on merge / failure

ingest mines PRs & test failures into facts; consolidate curates them — dedup, supersede, prune — so memory tracks the repo.

§03 · how it works
The AI already knows your codebase's rules — before it writes a line.

A skill is its method plus what it has learned about your repo. We read that straight out of your code — so the moment a skill is invoked, it already carries your house rules. Nothing has to break first.

We scan the codebase once and the rules it proves — stdlib-only, a fixed import style, how command-line tools are written — become memory. When any skill is invoked, that memory is inlined above its method automatically. No test ran. Nothing failed. No AI call. Re-run: ./demo/evolve.sh.

And it keeps getting sharper — by itself.

When the AI hits a rule it couldn't have guessed, the failed check teaches the memory one fact. You don't need to know any jargon:

1 · ask the ai

“Turn this page title into a URL.”

“My Cool Title”  →  my-cool-title

Wrong here. This codebase writes URLs with underscores (my_cool_title), not dashes. The AI had no way to know that house rule — so it broke the app.

2 · it learns — by itself

The failed check teaches the memory one fact.

“in this codebase, URLs use underscores, not dashes”

No human corrected it. No second AI. Zero AI calls. The assistant's own instructions are never edited — only what it knows grows.

3 · ask again — same ai

Same request, moments later.

“My Cool Title”  →  my_cool_title

Right now — and it stays right, for you and every teammate. It learned your codebase's rule once, from one failure.

One skill picking up three house rules from failed checks — slug (the URL form of a title), status (which error number bad input returns — the AI guessed 400; this app uses 422), timestamp (the date format) — and its instruction file is never edited. ./demo/skill-evolves.sh.

We re-ran this with real Claude agents. Without the learned context they got 1 of 3 right; with it, 3 of 3 — the actual receipts, every command re-runnable:

Same agent, same task — the only difference is whether it had the recalled rule. Left column has no learned context (1/3); right is the same AI with it (3/3). Re-run: python3 demo/ab.py.

§04 · team & org
One shared brain. Every agent on the team evolves with it.

Point every repo and teammate at one shared memory. It self-evolves from your merged pull requests — so when one engineer's PR lands a convention, every teammate's agents recall it on the next skill they invoke. The skills are never rewritten; the org's shared memory is. Real HTTPS, per-repo tokens, one small box.

# every dev points their skills at the org's shared memory
export YUNAKI_SUPERMEM_URL=https://your-supermem.example
export YUNAKI_SUPERMEM_TOKEN=<your repo token>
# strictly additive + fail-open — offline falls back to local context

The loop, end to end: a teammate merges a PR, GitHub fires the webhook, and the shared memory rebuilds itself from the merged code — here the store goes from 0 to 4 conventions, re-scanned on the merge. Every teammate then recalls the refreshed rules over HTTPS on the next skill they invoke. No rewrites, no model calls.

And it's really deployed — the same shared memory answering live over HTTPS on one small box, per-repo tokens, padlock and all.

§05 · get started
One command. No build step.
# wire it onto every skill (idempotent, reversible)
git clone <repo> && cd yunaki-skills
./install.sh

# seed + self-evolve the super memory from your PRs
./ingest_pr.py                 # auto-detects the repo
./ingest_pr.py --install-git-hook   # re-ingest + curate on every pull/merge

stdlib-only. markdown in, markdown out. no JSON, no skill rewriting.
